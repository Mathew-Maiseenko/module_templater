# BananaCat Module Templater

Утилита командной строки (CLI) для создания модулей проекта из предопределенных шаблонов.

## Быстрый старт

Выполните следующую команду в корневой директории вашего проекта:

```bash
npx bananacat-module-templater <ИмяТипаМодуля> <НазваниеМодуля> [путь] --var <имя_переменной>=<значение>
```

## API

### Команда

Основная команда для генерации модуля:

```bash
npx bananacat-module-templater <ModuleTypeName> <ModuleTitle> [path] [options]
```

### Аргументы

1.  `<ModuleTypeName>` (обязательный)

    - **Описание:** Тип создаваемого модуля. Это имя должно соответствовать названию поддиректории в `banana_templates.local`.
    - **Пример:** `module`, `feature`, `widget`

2.  `<ModuleTitle>` (обязательный)

    - **Описание:** Название вашего нового модуля. Это значение автоматически становится доступным в шаблонах как зарезервированная переменная `__MODULE_TITLE__`.
    - **Пример:** `MyAwesomeComponent`, `UserProfile`

3.  `[path]` (необязательный)
    - **Описание:** Путь к проекту, к котором необходимо создать модуль.
    - **По умолчанию:** Текущая рабочая директория (`process.cwd()`).

### Опции

- `--var <name=value>`
  - **Описание:** Позволяет передавать кастомные переменные в шаблоны. Можно использовать несколько раз для передачи нескольких переменных.
  - **Пример:** `--var componentName=Aboba --var cssStyle=bold`

## Конфигурация шаблонов (`banana_templates.local`)

Для работы утилиты в корне вашего проекта должна находиться директория `banana_templates.local`.

### Структура директории

```
.
├── banana_templates.local/
│   ├── variables.config.json
│   └── <ИмяТипаМодуля>/
│       ├── ИмяПапкиМодуля1
│				│		├── ИмяФалаМодуля1.tsx
│       │		└── styles.css
│				├── {{__MODULE_TITLE__}} -> заменится на <НазваниеМодуля>
│				│		├── ИмяФалаМодуля1.tsx
│       │		└── styles.css
│       └── styles.css
└── ...
```

- `banana_templates.local/`: Корневая папка для всех шаблонов.
- `variables.config.json`: (Опционально) JSON-файл для определения глобальных переменных, доступных во всех шаблонах.
- `<ИмяТипаМодуля>/`: Директория, имя которой соответствует аргументу `<ModuleTypeName>`. Содержит файлы и папки шаблона для одного типа модуля.

### Шаблонные переменные

Вы можете использовать переменные для динамической генерации контента в файлах и их названиях.

- **Синтаксис:** `{{__ИМЯ_ПЕРЕМЕННОЙ__}}`

Утилита заменит эти плейсхолдеры на соответствующие значения при генерации модуля.

**Пример:**

Если у вас есть файл `banana_templates.local/module/{{__MODULE_TITLE__}}.ts`, и вы запускаете команду:

```bash
npx bananacat-module-templater module MyComponent
```

Будет создан файл `MyComponent.ts`.

### Зарезервированная переменная `__MODULE_TITLE__`

- `__MODULE_TITLE__` — это специальная, зарезервированная переменная.
- Ее значение всегда равно второму аргументу командной строки — `<ModuleTitle>`.
- Это удобно для именования основного файла или директории модуля в соответствии с его названием.

### Файл `variables.config.json`

Этот файл позволяет определить переменные, которые будут доступны во всех шаблонах. Переменные, переданные через опцию `--var`, имеют более высокий приоритет и перезапишут значения из этого файла при совпадении имен.

**Пример `variables.config.json`:**

```json
{
  "author": "John Doe",
  "defaultColor": "blue"
}
```
